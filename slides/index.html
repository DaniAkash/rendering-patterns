What is Edge?
Servers that are hosted close to the users

Distributed across the Globe

Eg: CDNs

  • AWS Cloudfront
  • Cloudflare CDN
What is SSR?
React powered Single Page Applications
Client side rendering
Client side rendering is great
  • Only need to host static files
  • Excellent UX once the page has been loaded
  • Great developer experience
  • Hiccups with SEO
  • Hosted on the Edge
Client side rendering is not so great
  • Initial page load leads to layout shift
  • Bad cold start times
  • Too much JS payload for the browsers over time
Introducing Server Rendering
Static Site Generation (SSG)

SSG is great for

  • Content rich websites
  • Extremely good SEO
  • Very Reliable websites
  • Hosted on the Edge

Downsides of SSG

  • Can't display dynamic content with same advantages
Server Side Rendering (SSR)
1. Renders the first HTML on the server
2. Lets client takeover on the browser

SSR challenges

  • Needs an active Node.js Server
  • Large Traffic will lead to the Server's death
  • Node.js Server doesn't live on the edge
Solving the scaling problem with serverless
Serverless Node has cold start & no edge location
Edge Compute
Very Lightweight Node-like environments that run close to the users
  • Deno (Deno Deploy, Netlify Functions)
  • Bun (Oven)
  • Workerd (Cloudflare Workers, Vercel)
Edge Environments lack native APIs like fs, tls etc.
Working with databases will be challenging as central data means more latency
